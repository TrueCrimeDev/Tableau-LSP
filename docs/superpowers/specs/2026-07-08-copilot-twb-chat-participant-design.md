# Copilot Chat Participant for .twb Workbooks ‚Äî Design

**Date:** 2026-07-08
**Status:** Approved (user-approved section by section in session)

---

## Summary

Add a `@tableau` chat participant to the extension so GitHub Copilot Chat can
answer questions about the data inside the active `.twb` workbook. The
participant parses the workbook with the extension's existing parsers, builds a
bounded structured digest, and sends it ‚Äî together with a .twb-anatomy agent
prompt ‚Äî to Copilot's language model, streaming the answer back into chat.

Approach chosen: **LM-backed, digest-in-prompt** (approach A). Tool-calling
(approach B) was considered and deferred; the digest builder can become tool
outputs later if workbooks outgrow a single prompt.

---

## Components

### `src/chat/twbPrimer.ts`

Exports `TWB_AGENT_PRIMER`, the agent prompt, written in the conventions of the
user's Prompts repo (`*.system-prompt.md`): a root `<TABLEAU_AGENT_INSTRUCTION>`
tag with sections:

- `<role>` ‚Äî Tableau workbook XML analyst; answers questions about and guides
  edits to .twb files using the supplied workbook digest.
- `<twb_anatomy>` ‚Äî workbook section order (`document-format-change-manifest ‚Üí
  preferences ‚Üí style ‚Üí datasources ‚Üí worksheets ‚Üí windows ‚Üí thumbnails`) and
  what lives in each.
- `<border_model>` ‚Äî the four border slots (workbook `<style>`, worksheet
  `<table><style>` style-rules `cell`/`header`/`pane`/`table-div`, pane-level
  styles, `<format>` grammar), div-level/stroke-color dividers, and the
  inheritance rule: explicit `none`/`0` nodes are required to suppress
  inherited/default borders ‚Äî absence of a node means the default applies.
- `<calculations>` ‚Äî `[Calculation_xxx]` internal names resolve to captions;
  formulas live on `<calculation formula='‚Ä¶'>` inside `<column>` elements.
- `<thumbnails>` ‚Äî `<thumbnail name='SHEET' height width>` holding base64 PNG,
  keyed to worksheets by name, regenerated by Tableau on save.
- `<edit_guidance>` ‚Äî advising edits safely: neutralise border nodes
  (`none`/`0`) rather than delete, keep XML well-formed, expect Tableau to
  rewrite formatting sections on resave, recommend a backup before hand-edits.
- `<answer_rules>` ‚Äî voice rules derived from Writing/WritingRules: lead with
  the answer, cite worksheet names, plain words, no filler openings, say "not
  present in this workbook" or `[VERIFY]` rather than inventing details.

### `src/chat/workbookDigest.ts`

Pure module (no `vscode` import ‚Äî unit-testable like `formatStripper`):

```typescript
export type DigestFocus = 'borders' | 'calcs' | 'fields';
export function buildWorkbookDigest(xml: string, focus?: DigestFocus): string;
```

Reuses `cleanXmlContent` + `resolveNames` (src/extract) for calculation name
resolution and `scanFormattingXml` (src/parsers/formatStripper) for the
formatting overview. Emits markdown:

- Worksheet inventory (names).
- Workbook-level `<style>` summary (including FCP-mangled variants).
- Per-worksheet `<table><style>` style-rules with their `<format>` nodes.
- Calculations with captions and resolved formulas (caps: 50 calcs, 200-char
  formulas in the default view; full formulas under `/calcs` focus).
- Datasource fields (caption, datatype).
- Thumbnail inventory (name, dimensions, approx decoded KB ‚Äî never the base64).
- Formatting scan (border/bold/font-size/font-color counts + values).

Focus narrows output: `borders` ‚Üí full style detail only; `calcs` ‚Üí full
calculations only; `fields` ‚Üí fields only. Hard cap ~12k chars with an explicit
`[digest truncated‚Ä¶]` note appended so the model knows it saw a partial view.

### `src/chat/tableauChatParticipant.ts`

Thin VS Code layer plus testable core:

- `composeTableauMessages(xml, prompt, command)` ‚Äî pure: returns the two
  message strings (primer+digest, question ‚Äî default question per command when
  the user typed none).
- `describeChatError(err)` ‚Äî pure: maps `LanguageModelError`-shaped failures to
  short user-facing markdown.
- `registerTableauChatParticipant(context)` ‚Äî called from `activate()`; guards
  on `vscode.chat?.createChatParticipant` so older hosts no-op. Registers id
  `tableau-language-support.tableau`, icon `images/tableau2.svg`, followups for
  the three commands.
- Handler flow: resolve workbook (active editor `.twb` ‚Üí most recent `.twb`
  tab), read buffer-first (open TextDocument over disk), build messages, send
  via `request.model.sendRequest`, stream fragments to `stream.markdown`.

### `package.json`

- `contributes.chatParticipants`: name `tableau`, description, `isSticky`,
  commands `borders`, `calcs`, `fields`.
- `engines.vscode` and `@types/vscode`: `^1.80.0` ‚Üí `^1.95.0` (Chat + LM APIs
  stable). This drops extension updates for users on VS Code < 1.95.

## Error handling

- No workbook open / only `.twbx`: explanatory markdown, no LM call.
- No model (no Copilot access): plain message, no throw.
- `LanguageModelError` (quota, filtered): short message via `describeChatError`.
- Digest overflow: truncation note in-band.
- Cancellation token passed through to the LM request.

## Testing

- `src/tests/unit/workbookDigest.test.ts` ‚Äî inventory, style rules, calc
  resolution, field list, thumbnail inventory without base64 leakage, focus
  narrowing, truncation cap.
- `src/tests/unit/tableauChat.test.ts` ‚Äî primer section presence; message
  composition (primer first, digest included, question last, command defaults);
  error mapping.
- Handler adapter is exercised by typecheck + manual checklist: `@tableau`
  appears in chat, streams an answer, `/borders` works, no-workbook path
  answers gracefully. (The chat surface itself cannot run under jest.)

## Out of scope

- `.twbx` support (rejected with a message, matching the sidebar).
- Tool-calling (`vscode.lm.registerTool`) ‚Äî future evolution.
- Editing the workbook from chat; the participant analyses and advises only.
