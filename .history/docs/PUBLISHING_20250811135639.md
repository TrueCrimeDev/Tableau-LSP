# Publishing Guide

This document outlines local and CI publishing flows for the Tableau Language Support extension.

## One-Command CI Release (Recommended)

Requires GitHub Actions workflow and `VSCE_PAT` secret configured.

1. Update code and `CHANGELOG.md`.
2. Bump version & create tag:
   ```bash
   npm version patch -m "chore: release v%s"
   git push && git push --tags
   ```
3. GitHub Actions workflow builds and publishes automatically.

## Local Publish (Manual)

1. Set Marketplace token in shell:
   ```powershell
   $env:VSCE_PAT = 'your-marketplace-pat'
   # or persist
   setx VSCE_PAT "your-marketplace-pat"
   ```
2. Package:
   ```powershell
   npm run package
   ```
3. Publish:
   ```powershell
   npm run publish
   ```

Or use VS Code tasks:
- Run: `Package VSIX (vsce package)`
- Run: `Publish latest VSIX to VS Code Marketplace`

## Tag-Triggered Publish Without Version Bump Script
```powershell
git tag v1.5.2
git push origin v1.5.2
```

## Manual Workflow Dispatch
If `gh` CLI installed:
```powershell
gh workflow run "Publish VS Code Extension" --ref main
gh run view --log
```

## Token Scope
Create a VS Code Marketplace (Azure DevOps) Personal Access Token with: `Marketplace (Publish)` scope. Add to GitHub repo secrets as `VSCE_PAT`.

## Common Pitfalls
- Missing `VSCE_PAT`: publishing fails immediately.
- Duplicate version: `--skip-duplicate` avoids error but nothing new is published.
- Large VSIX: Ensure `.vscodeignore` excludes tests and large assets.

## Minimal Checklist
- [ ] Code changes merged
- [ ] `CHANGELOG.md` updated
- [ ] `package.json` version bumped
- [ ] Tag pushed
- [ ] CI workflow green

## .vscodeignore Suggestion

```

**/*.map
.vscode/
.git/
**/*.log
images/*.psd
```
